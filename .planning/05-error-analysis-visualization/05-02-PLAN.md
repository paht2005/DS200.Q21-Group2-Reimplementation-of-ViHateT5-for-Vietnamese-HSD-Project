Rewrite `scripts/run_error_analysis.py` completely. Follow the `scripts/run_ensemble.py` structural pattern.

**Imports and sys.path:** Same pattern as run_ensemble.py — sys.path.insert for src/, import json, os, argparse, numpy, pandas, torch. Import from src: `from error_analysis import run_full_error_analysis, mcnemar_report, plot_combined_comparison`, `from data_loader import load_vihsd, load_victsd`.

**Helper functions to include:**

1. `detect_model_type(model_path)` — Copy from run_ensemble.py (reads config.json, checks architectures for "T5" or "ForSequenceClassification", fallback on model_type field). Returns tuple (type_str, config_dict).

2. `get_all_finetuned_models()` — Discovers models in `models/` directory, excludes entries containing "pretrain". Returns list of relative paths like `["models/vit5_finetune_balanced", ...]`.

3. `predict_t5(model_path, texts, task, device, batch_size)` — Load T5ForConditionalGeneration + AutoTokenizer from model_path to device. Generate predictions with task prefix ("hate-speech-detection: " for vihsd, "toxic-speech-detection: " for victsd). Use num_beams=1, do_sample=False, max_length=64. Map text output to integers with case-insensitive label_map: {"clean":0, "offensive":1, "hate":2} for vihsd, {"none":0, "toxic":1} for victsd. Default unknown to most-common class (0). Delete model+tokenizer and call torch.mps.empty_cache() after predictions. Return list[int].

4. `predict_bert(model_path, texts, device, batch_size)` — Load AutoModelForSequenceClassification + AutoTokenizer from model_path. Tokenize in batches (max_length=256, padding=True, truncation=True). Forward pass → argmax(logits, dim=1). Delete model+tokenizer + empty_cache. Return list[int] of raw label indices.

5. `get_predictions(model_path, texts, task, device, batch_size)` — Unified dispatcher. Calls detect_model_type. If "t5" → predict_t5. If "bert" → predict_bert, then apply label_remap {0:0, 1:2} when task=="vihsd" and model has 2 labels (check config num_labels or id2label length). Return list[int].

6. `load_cached_predictions(model_name, task)` — Check if `results/analysis/{model_name}_{task}_predictions.csv` exists. If yes, read and return pred_label column as list[int]. If no, return None.

7. `save_predictions(model_name, task, texts, true_labels, pred_labels)` — Save DataFrame with columns: text, true_label, pred_label to `results/analysis/{model_name}_{task}_predictions.csv`.

**Argparse (parse_args function):**
- `--models` nargs="+" — specific model paths
- `--all-models` store_true — use all fine-tuned models (default behavior when --models not specified)
- `--task` choices=["vihsd", "victsd", "both"], default="both"
- `--batch-size` int, default=8
- `--no-cache` store_true — force re-run inference even if cached predictions exist
- `--output-dir` str, default="results" — base output directory

**Main function flow:**

Step [1/7] Device detection (MPS-aware, same as run_ensemble.py).

Step [2/7] Resolve model list: if --models provided use those, else call get_all_finetuned_models(). Print model list.

Step [3/7] Load test data: call load_vihsd() and/or load_victsd() based on --task. Extract test texts (vihsd: "free_text" column, victsd: "Comment" column) and true labels (vihsd: "label_id", victsd: "Toxicity").

Step [4/7] Generate/load predictions for each model on each task. For each model: derive model_name (basename of path, e.g., "vit5_finetune_balanced"). Check cache unless --no-cache. If not cached, call get_predictions and save_predictions. Store in dict: predictions[model_name][task] = list[int]. Print progress: "[4/7] Predictions: {model_name} on {task} — {cached|computed}".

Step [5/7] Run per-model error analysis. For each model_name + task, call `run_full_error_analysis` with model_name parameter set. This produces per-model confusion matrices, F1 charts, error distribution, failure cases, bootstrap CI — all with model-specific filenames.

Step [6/7] McNemar's tests. Define test_pairs as star pattern: baseline="vit5_finetune_balanced", challengers = all other 4 models. For each task, call `mcnemar_report(test_pairs, true_labels, predictions_dict_for_task, task_name, filename=f"mcnemar_results_{task}.csv")`. Then attempt best-vs-ensemble: load ensemble predictions from Phase 4 if available (check results/analysis for ensemble prediction CSV or recompute from ensemble_results.csv aggregate — if per-sample preds not available, skip this pair with a printed warning).

Step [7/7] Combined comparison chart. Build a DataFrame (model_name, task, macro_f1) from all model predictions + ensemble results (read ensemble Macro_F1 from results/ensemble_results.csv where Method=="weighted" or best). Call `plot_combined_comparison(results_df)`. Print summary table.

Final: Print file listing of results/images/ and results/analysis/ with sizes.
